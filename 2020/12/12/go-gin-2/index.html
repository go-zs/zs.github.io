golang系列(26)--gin源码阅读(2)

发表于 2020-12-12 更新于 2021-04-20 分类于 golang

Context是gin框架中最重要的概念。

结构

Context是一个结构体，源码如下。

gin.Context实现了标准库中Context接口。

Bind

Context支持非常多的数据绑定方式。

最常用的Bind方法支持自动通过请求方法和请求头，自动参数绑定。

其它方法基本都是通过指定数据类型进行绑定。

Binding

对于不同的数据类型处理，gin定义了Binding这个接口，这样扩展数据类型只需要实现Binding接口。

看下最常见的json的数据绑定的实现。

通过+build方式条件编译实现了自定义的json解析方式。使用go build main.go的编译方式使用的是标准库encoding/json，而使用go build -tags=jsoniter main.go进行编译则会使用json-iterator来进行json的处理。

Params, Query

Params获取可以获取url路径中的参数，Query则是可以解析url问号后面附带的请求参数。

Keys

Keys提供了Set和Get这两个方法可以帮助我们传递线程安全的变量。

由于返回值是interface{}，gin也提供了很多API帮助我们从map中读取指定类型的数据。

Copy

Context并不是并发安全的，所以如果我们需要多个goroutine处理Context，需要用Copy方法复制一份。
